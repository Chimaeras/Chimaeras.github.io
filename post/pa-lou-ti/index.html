爬楼梯

2021-08-28

假设你正在爬楼梯。需要 n 阶你才能到达楼顶。

每次你可以爬 1 或 2 个台阶。你有多少种不同的方法可以爬到楼顶呢？

注意：给定 n 是一个正整数。

  • 方法1：动态规划

通过分析可以得到，最后一步可能跨了一级台阶，也可能垮了两级台阶。所以通项公式为：f（x）=f（x-1）+f（x-2）。

初始条件

  • f（0）=1：从第0级到第0级，只有一种方案。
  • f（1）=1：从第0级到第1级，只有一种方案，即跨一级台阶。
  • f（2）=2：从第0级到第2级，有两种方案，即跨两级台阶或者两次跨一级台阶。
class Solution {
    public int climbStairs ( int n ) {
        int p = 0, q = 0, r = 1;
        for ( int i = 1; i <= n; ++i ) {
            p = q;
            q = r;
            r = p + q;
        }
        return r;
    }
}

  • 方法2：数学推导

根据递推公式我们可以尝试得到特征方程，对于一个数列：

X（n+2）=c1∗X（n+1）+c2∗X（n）X（n+2）=c1*X（n+1）+c2*X（n） X（n+2）=c1∗X（n+1）+c2∗X（n）

假设有r，s使得

X（n+2）−r∗X（n+1）=s∗[X（n+1）−r∗X（n）]X（n+2）-r*X（n+1）=s*[X（n+1）-r*X（n）] X（n+2）−r∗X（n+1）=s∗[X（n+1）−r∗X（n）]

所以有：

X(n+2)=(s+r)∗X(n+1)−sr∗XnX(n+2)=(s+r)*X(n+1)-sr*Xn X(n+2)=(s+r)∗X(n+1)−sr∗Xn

可以得到

c1=s+r;c2=−s∗r;c_1=s+r;c_2=-s*r; c1​=s+r;c2​=−s∗r;

消去s导出特征方程，即

r2=c1∗r+c2r^2=c_1*r+c_2 r2=c1​∗r+c2​

将递推公式中常量代入，得到特征方程。

运用上述方法对公式进行推导得：

x2=x+1x^2=x+1 x2=x+1

通过求根公式可知：

x1=(1+5)/2;x2=(1−5)/2x_1=(1+\sqrt5)/2;x_2=(1-\sqrt5)/2 x1​=(1+5​)/2;x2​=(1−5​)/2

设通解为

f(n)=c1∗x1n+c2∗x2nf(n)=c_1*x_1^n+c_2*x_2^n f(n)=c1​∗x1n​+c2​∗x2n​

代入初始条件f（1）=1，f（2）=1，得：

c1=1/5;c2=−1/5c_1=1/\sqrt5;c_2=-1/\sqrt5 c1​=1/5​;c2​=−1/5​

则完整通项公式为：

f(n)=1/5∗[((1+5)/2)n−((1−5)/2)n]f(n)=1/\sqrt5*[((1+\sqrt5)/2)^n-((1-\sqrt5)/2)^n] f(n)=1/5​∗[((1+5​)/2)n−((1−5​)/2)n]

public class Solution {
    public int climbStairs ( int n ) {
        double sqrt5 = Math.sqrt( 5 );
        double fibn = Math.pow( ( 1 + sqrt5 ) / 2, n + 1 ) - Math.pow( ( 1 - sqrt5 ) / 2, n + 1 );
        return ( int ) Math.round( fibn / sqrt5 );
    }
}

总结

  • 当n比较小的时候，可以通过递归法来求解，时间复杂度为O（2^n）
  • 一般情况下，考虑迭代法或备忘录法，时间复杂度为O（n）
  • 基于上一条，我们可以只用三个变量来维护三个状态，即滚动数组思想，可将空间复杂度优化到O（1）
  • 当n很大时，可以考虑【矩阵快速幂】将算法优化到O（logn）
  • 通过数学推导得出通项公式，可直接求结果，可能产生精度误差
